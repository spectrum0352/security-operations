This section provides an overview of the licensing requirements for Microsoft Defender XDR, Microsoft Entra, Microsoft Purview, and related Microsoft security products.

# Microsoft Defender Licensing

## Microsoft Defender for Endpoint

### Microsoft Defender for Endpoint Plan 1 (P1)

Microsoft Defender for Endpoint Plan 1 provides core endpoint protection capabilities, including:

- Next-Generation Protection (NGAV)
- Attack Surface Reduction (ASR)
- Device Control
- Firewall Management
- Web Protection
- Manual Response Actions
- Basic Endpoint Detection and Response (EDR)

Suitable for organizations requiring essential endpoint security.

---

### Microsoft Defender for Endpoint Plan 2 (P2)

Microsoft Defender for Endpoint Plan 2 includes all P1 capabilities plus advanced enterprise security features:

- Full Endpoint Detection and Response (EDR)
- Threat & Vulnerability Management (TVM)
- Automated Investigation and Remediation (AIR)
- Advanced Hunting
- Live Response
- Device Discovery
- Threat Analytics
- Endpoint Isolation
- File Collection
- Custom Detection Rules
- Microsoft Security Copilot integration (where licensed)

Recommended for enterprise security operations centers (SOC).

---

### Microsoft Defender for Servers

Server protection is licensed through **Microsoft Defender for Cloud**.

Supported server operating systems include:

- Windows Server
- Linux Server
- Azure VMs
- AWS EC2
- Google Cloud Compute Engine
- On-premises servers (via Azure Arc)

Available plans:

- Defender for Servers Plan 1
- Defender for Servers Plan 2 (includes Defender for Endpoint P2 capabilities)

# How to Check Assigned Licenses

## Microsoft 365 Admin Center

Navigate to:

```
https://admin.microsoft.com

Billing
└── Licenses
```

or

```
Users
└── Active Users
    └── Licenses and Apps
```

Required roles:

- Global Administrator
- Billing Administrator
- License Administrator
- Global Reader (read-only)

---

## Microsoft Entra Admin Center

Navigate to:

```
https://entra.microsoft.com

Identity
└── Users
    └── Assigned Licenses
```

This allows administrators to verify licenses assigned to individual users.

---

## Microsoft Defender Portal

Navigate to:

```
https://security.microsoft.com

Settings
└── Endpoints
```

Some Defender features display licensing status and availability based on your tenant subscriptions.

# Licensing Best Practices

- Verify licenses before enabling advanced security features.
- Assign licenses using Microsoft Entra security groups where possible.
- Use Microsoft Entra ID P2 for organizations implementing Zero Trust.
- Use Microsoft Defender for Endpoint P2 for enterprise SOC deployments.
- Regularly audit unused licenses to optimize licensing costs.
- Review licensing requirements whenever enabling new Microsoft Defender, Entra, or Purview capabilities, as some features require additional subscriptions.
