[Blocking attack indicators](https://github.com/sandeepj0352/Microsoft-Defender/wiki/Blocking-attack-indicators) [playbook](https://github.com/sandeepj0352/Microsoft-Defender/commit/80050c7314bc93f3f2395680c0f12c7f285cb8cf)

blocking malicious ip - Firewall Rule Update: Automatically updates Azure Firewall rules based on incident details (e.g., blocking IPs).

conditional access response - Azure AD Conditional Access Response: Automatically applies conditional access policies to block suspicious user behavior.

Data Leakage Prevention: Detects and alerts on unauthorized file downloads or sensitive data transfers.

Disabling compromised accounts

File Integrity Monitoring (FIM): Monitors changes to sensitive files or folders and generates alerts or takes actions.

Geo-Location Alerting: Tracks logins or activities from suspicious geographic locations and sends alerts

incident enrichment - Fetches additional details (IP reputation, threat intelligence) for incidents to assist in analysis

isolate compromised endpoints - Isolating endpoints Auto-Remediation: Automatically takes containment actions like isolating VMs

ITSM Ticket Integration: Creates tickets in ITSM systems like ServiceNow or Jira for tracking and workflow management.

Notification Playbook: Sends notifications to email, Microsoft Teams, or Slack with incident details when triggered.

Phishing Response: Searches for phishing emails, quarantines them, and notifies recipients.

Ransomware Response: Isolates affected devices, disables accounts, and initiates recovery processes during ransomware attacks.

SIEM Health Monitoring: Monitors Sentinel's health, such as ingestion latencies, data connectors, and analytic rules

Malware Analysis: Submits suspicious files or hashes to analysis platforms like VirusTotal or Hybrid Analysis.

Threat Intelligence Lookup: Enriches incidents by querying threat intelligence platforms (e.g., VirusTotal, AlienVault)
